## Feature 1: RAG 多查询扩展

### What it does

When the agent routes to `retrieve_knowledge`, instead of running a single vector search, it:
1. Calls LLM once to rewrite the original query into 2–3 semantically equivalent sub-queries
2. Runs `retriever.retrieve()` for each sub-query independently
3. Deduplicates results by `chunk_id`, sorts by score descending, returns merged top-k

### Files changed

| File | Change |
|---|---|
| `app/agent/nodes/act_retrieve.py` | Add `_expand_queries()` helper using `ModelService`; call it before retrieval |

No changes to `retriever.py`, DAG, or state schema.

### Query expansion prompt (inlined in act_retrieve.py)

```
Given the user query below, generate {N} alternative phrasings that have the same meaning
but use different words. Output as a JSON array of strings. Do not explain.

Query: {query}
```

N = 2 (original + 2 expansions = 3 total queries). If LLM call fails or returns malformed JSON,
fall back silently to single-query retrieval (original behavior).

### Data flow

```
act_retrieve.__call__()
  → _expand_queries(query)          # 1 LLM call, returns [q1, q2, q3]
  → retriever.retrieve(q1)          # FAISS search
  → retriever.retrieve(q2)
  → retriever.retrieve(q3)
  → _merge_dedupe(results)          # dedup by chunk_id, sort by score desc
  → state["retrieved_chunks"] = top_k merged results
```

### Fallback

If `_expand_queries()` raises any exception, log a warning and proceed with `[original_query]` only.

## Feature 2: 家长侧摘要生成工具

### What it does

A new LangGraph tool `generate_parent_summary` available in parent mode. When a parent asks
"帮我总结一下孩子最近的情况" (or similar), the reason node selects this tool. The tool:
1. Reads **all** episodic + semantic memory entries for `default_child` from SQLite
2. Passes the full content to LLM with a structured summary prompt
3. Returns a natural-language parent report covering: recent interests, frequent topics, learning observations

### Files changed

| File | Change |
|---|---|
| `app/tools/basic_tools.py` | Add `GenerateParentSummaryInput` schema; add `_langgraph_generate_parent_summary()` and `_generate_parent_summary()` methods; register tool in `_build_langgraph_tools()` |
| `app/prompts/tutor_prompts.py` | Add `build_parent_summary_prompt(memory_texts)` function |

No changes to DAG, AgentState, or other modes.

### Tool schema

```python
class GenerateParentSummaryInput(BaseModel):
    child_profile_id: str = Field(default="default_child")
```

### Memory query

```python
episodic_items = manager.store.list_items(user_id="default_child", memory_type="episodic", limit=10_000)
semantic_items = manager.store.list_items(user_id="default_child", memory_type="semantic", limit=10_000)
```

All entries, no filter. If both lists are empty, return a message indicating no child history found.

### Summary prompt structure

```
You are a child development assistant writing a report for a parent.
Below are memory records from the child's learning sessions with XiaoZhi.
Summarize in Chinese, covering:
1. 最近感兴趣的话题
2. 常问的问题类型
3. 学习观察与建议

Memory records:
{memory_texts}
```

### Tool registration

Registered in `_build_langgraph_tools()` alongside the existing three tools.
The reason prompt already instructs the model to choose tools — no prompt change needed
as long as the tool description is clear:

```
"Generate a summary report of the child's recent learning sessions for the parent,
based on stored memory. Use this when the parent asks for a child activity summary."
```

### Data flow

```
parent asks: "总结一下孩子最近的情况"
  → reason node selects generate_parent_summary tool
  → ToolNode calls _langgraph_generate_parent_summary(child_profile_id="default_child")
  → _generate_parent_summary():
      reads all episodic + semantic from SQLite
      calls LLM with build_parent_summary_prompt(texts)
      returns summary string
  → observe node summarizes tool result
  → respond node streams summary to parent
```

## Acceptance criteria

- [ ] No duplicate `chunk_id` in `retrieved_chunks`
- [ ] `workflow_trace` shows `act_retrieve` as before (no new node)

### 家长侧摘要生成
- [ ] Tool appears in the bound tool list for parent mode
- [ ] When SQLite has child memory, LLM generates a structured Chinese summary
- [ ] When SQLite has no child memory, returns a graceful "no history" message
- [ ] education / companion mode unaffected (tool exists but reason node won't call it)
